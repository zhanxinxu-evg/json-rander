# Edgen 图表需求指南

> 面向产品经理：如何提出新图表类型需求

---

## 概要

通用图表的数据完全由 AI 生成，前端负责纯渲染。产品只需描述清楚"要什么图表、展示什么信息、什么场景使用"，技术细节（Schema 定义、AI 填充规则、异常处理等）由开发团队负责。

**产品需要提供什么？**

| 交付物 | 说明 |
|---|---|
| **图表名称与用途** | 这个图表叫什么、解决什么分析场景 |
| **与现有图表的区别** | 为什么不能复用已有图表类型 |
| **展示信息描述** | 图表上应该展示哪些信息（维度、指标、标签等） |
| **使用场景** | 什么对话/分析场景下应该展示这个图表 |

> **注意：** 提需求前请先查阅 [chart-examples.md](./chart-examples.md) 确认已有的 21 种图表类型是否已覆盖你的需求。
>
> 如果组件需要实时数据（如行情价格、板块热力图），应提为**业务组件**需求，参考 [business-component-guide.md](./business-component-guide.md)。

---

## 需求模板

```markdown
## [图表名称]

### 用途
- 一句话描述：
- 典型分析场景（列出 2-3 个）：

### 与现有图表的区别
- 最相似的图表是哪个？为什么不能复用？

### 展示信息
- 图表上应该展示哪些信息：
- 数据量范围（最少几个数据点、建议几个、最多几个）：

### 大概样式
- 图表长什么样（文字描述、草图、截图或参考图均可）：

### 使用场景
AI 应该在什么情况下使用这个图表（列出 2-3 个典型对话场景）：
- "……"
- "……"
- "……"

不适合使用此图表的场景（列出 1-2 个容易混淆的场景）：
- "……" → 应该用 XX 图表
```

## 完整示例

### 示例 1：雷达图（RadarChart）

**用途**
- 一句话描述：在多个维度上对比实体的能力分布，以雷达网格形式直观展示各维度强弱
- 典型分析场景：AI 模型多维度能力对比、竞品产品特性对比、投资组合风险维度分析

**与现有图表的区别**

| 对比图表 | 区别 |
|---|---|
| BenchmarkHeatmap | Heatmap 是表格式色块，适合实体较多；雷达图是图形化网格，更直观但实体数量受限 |
| VersusChart | Versus 限定两个实体一对一对比；雷达图支持 1-3 个实体叠加 |
| BarChart | 柱状图是一维对比；雷达图展示多维度的"形状"差异 |

**展示信息**
- 图表上应该展示哪些信息：
  - 多个维度轴（如"推理"、"编码"、"创意写作"等）
  - 每个实体在各维度上的数值（构成一个多边形）
  - 多个实体叠加显示，方便对比形状差异
  - 可高亮某个实体
- 数据量范围：最少 3 个维度，建议 5-6 个，最多 8 个；实体 1-3 个

**大概样式**
- 从中心向外发散出多条轴线，每条轴代表一个维度
- 每个实体的各维度数值连成一个封闭多边形，覆盖在轴线之上
- 多个实体的多边形叠加展示，通过颜色区分，一眼可看出"形状"差异

**使用场景**
- "对比一下 GPT-4o 和 Claude 3.5 的各项能力"
- "分析一下 Tesla 在品牌力、技术、成本控制等方面的竞争力"
- "几个投资组合的风险维度对比"

不适合使用此图表的场景：
- 只对比 2 个维度 → 应该用 ComparisonBar
- 实体超过 3 个 → 应该用 BenchmarkHeatmap
- 纯排名 → 应该用 RankingBar
