# Edgen 业务组件需求指南

> 面向产品经理：如何提出业务组件需求

---

## 概要

业务组件与通用图表不同——**通用图表的数据由 AI 生成，业务组件的数据由前端从后端 API 获取**。产品只需描述清楚"要什么组件、解决什么场景"，技术细节（参数设计、Schema 定义、异常处理等）由开发团队负责。

业务组件分为两个子类：

| 子类 | 特征 | 典型场景 |
|---|---|---|
| **复杂组件** | 复用产品已有组件，或新设计带复杂交互的卡片；前端调 API 取实时数据后渲染 | 历史价格走势图、板块热力图 |
| **简单引导组件** | 单一行为——点击即跳转；不同跳转类型可以有不同卡片样式 | 跳转到个股详情、跳转到板块页、跳转到功能页 |

**产品需要提供什么？**

| 交付物 | 说明 |
|---|---|
| **组件名称与用途** | 这个组件叫什么、解决什么场景、为什么不能用通用图表 |
| **用户触发场景** | 用户会怎么问——列出典型的自然语言表述 |
| **期望的展示效果** | 组件大致长什么样、展示哪些信息（文字描述或截图均可） |

---

## 复杂组件 vs 简单引导组件

| 问题 | 选复杂组件 | 选简单引导组件 |
|---|---|---|
| 需要在组件内展示数据或图表吗？ | 是 | 否，只需引导用户跳转 |
| 用户需要在组件内进行交互吗？ | 是，如切换时间范围、切换指标 | 否，唯一操作就是点击跳转 |

## 完整示例

### 示例 1：历史价格走势图（复杂组件）

**用途**
- 一句话描述：展示某个资产在指定时间范围内的历史价格走势
- 为什么不能用通用图表：行情数据量大（数百个数据点）、需要实时、AI 无法精确提供每日价格

**用户场景**
- "看一下苹果最近的股价"
- "比特币过去一年的走势"
- "特斯拉上市以来的价格"
- "XX 的 K 线 / 行情 / 走势图"

**期望展示**
- 组件展示哪些信息：带时间轴的价格折线图，可查看具体日期的价格
- 交互能力：可切换时间范围（1天 / 1周 / 1月 / 3月 / 6月 / 1年 / 5年 / 全部）

---

### 示例 2：板块热力图（复杂组件）

**用途**
- 一句话描述：以色块矩阵展示某个市场各板块的实时涨跌情况，面积代表市值权重
- 为什么不能用通用图表：板块数量多（几十个）、数据需实时、每个色块需包含涨跌幅/市值等多维信息

**用户场景**
- "今天美股各板块表现怎么样"
- "看一下加密货币市场热力图"
- "港股哪些板块在涨"
- "市场概览"、"大盘全景"

**期望展示**
- 组件展示哪些信息：矩形色块矩阵，红涨绿跌（或绿涨红跌），面积代表市值权重，颜色深浅代表涨跌幅度
- 交互能力：可切换度量方式（日涨跌 / 周涨跌 / 月涨跌），支持的市场包括美股、港股、加密货币

---

### 示例 3：个股详情跳转（简单引导组件）

**用途**
- 一句话描述：点击后跳转到某个资产的详情页

**用户场景**
- "帮我打开苹果的详情页"
- "看一下特斯拉的基本面"
- AI 主动触发：用户聊到某只股票时，附上一张跳转卡片方便用户查看详情

**卡片样式**
- 卡片上显示哪些信息：资产名称、当前价格、涨跌幅
